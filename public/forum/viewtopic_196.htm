Re: [Question] OllyDbg feature

by random » Sat Jul 14, 2012 6:58 pm

You can't set a breakpoint on if the value '121212121212' happens to enter memory, but you can break at any point and search for your serial. The way you do it is right click in the disassembly window and choose "binary" -> "Search" and enter your serial. You may have to try both ASCII and unicode, but this searches the entire memory space. You can also open the Memory Map window, right click on the area the main app is in and select "Search". This does the same thing but limits it to a certain section of memory.

Re: [Question] OllyDbg feature

by random » Sat Jul 14, 2012 8:14 pm

this is true. You must find out where the serial is processed. It's probably a callback if it's VC++. Then you can set your BP while the serial is still in memory.

Try this: BP on wherever the call into the kernel or user23 dll is (like DialogBox, GetDlgItem, whatever). Step in one time until you are in the windows API. The nmae of the API will be in the title of the CPU window. Open the memory map window. Right click on wherever your app's code resides (usually the text section) and choose place breakpoint on memory access. Then run the app. It should break as soon as the callback is called and this should be the area the serial is processed. If this doesn't work then you've got something a little tougher, and without knowing exactly what you're doing, I can't be much help.
