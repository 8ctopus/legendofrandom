Gaius Baltar wrote:What is this tool about? (EVlog)
(I googled on it but there are lots of mixed results so ...)

Random is correct, it is a tool that allows you to see the Environment Variables that an Armadillo protected program is using.

It's very clever, and very useful.

If you have an application that you identify is being protected by a version of Armadillo (using a PE Packer Identifier) then you can use this utility to launch that application and identify the variables being used - and their values.

Just knowing the values of some of the variables can provide insights into the protection logic being used. But once you un-pack the application (i.e. remove the Armadillo protection) you may find that you have also removed a number of the environment variables that were being set. In some cases, the application will look for these environment variables - and if they cannot be found - the developer concludes that something is amiss and decides to exit or behave differently.

Of course it is possible to identify these variable without this utility, but the utility gives you a heads up and also provides you with the values of the variables without having to trace the packed version of the application as well to see where the variables are set.
