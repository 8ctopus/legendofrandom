Protection Schemes w/Threads

by simplicity » Tue Dec 04, 2012 6:56 pm

Hello. I am having a difficult time trying to reverse a protection scheme that spawns a new thread. I'll try to make this as concise as possible.

1. I start stepping the program

2. Upon being asked for the serial, if the end-user opts to enter one, it spawns a new thread

3. I can look in the log window, and get the address of the .dll where the new threads starts (only after the thread is terminated)

4. I place breakpoint at that address (it's a .dll for the program)

5. I try registration scheme again, and I break at the previously set breakpoint

6. Now here is where I'm having problems. Seems like I am tracing through various .dll's (ole32.dll, shlwap.dll, and several others) forever. I've been tracing for days, and I have yet to see the string I have inputted or reach the end of the routine. What is making it difficult for me (I think) is ASLR. It will never break on the same spot twice, so every time I restart my debugger, I have to manually step through all of this code again.

Maybe this code has already broken my brain, but this is my first "real world" one and I'd like to know - does this sound right and I need to suck it up and keep going, or have I overcomplicated this process, taken a wrong turn and I'm driving in circles?

Any input would be appreciated. Thanks for reading.

quick example, pause ollydbg when registration dialog is displayed, get the handle from ollydbg ->view->windows.
look at SendMessage args
Code: Select all
0012FB04   0040144D  /CALL to SendMessageA from 00401448
0012FB08   0006023E  |hWnd = 6023E            ; edit1   
0012FB0C   0000000D  |Message = WM_GETTEXT      
0012FB10   00000019  |Count = 19 (25.)         ; includes null terminator
0012FB14   004045EA  \Buffer = 004045EA         ; destination

so bp SendMessageA [esp+4]==6023e && [esp+8]==0D && [esp+0c]==19
