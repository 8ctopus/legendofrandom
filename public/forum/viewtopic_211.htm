Re: Hiding API calls in C++ code

by nwokiller » Sun Nov 18, 2012 12:22 am

Some protectors do this by "emulating" API's which basically means they copy all the code for an API and execute it from the packer's section or another section. Another way is to load the API address into a register and then call it (this would hide it from "Intermodular calls in Olly")... Don't know how you would do this in C++ but the ideas are there.
