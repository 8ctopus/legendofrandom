My technique was as follows:

  • First find the applicable code section - that was easy since even searching for text strings found it. The applicable jmp/no jump comparison was also obvious so patching the exe woudl of course be simple, but we needed a key for the challenge
  • stare at the code carefully to determine how it worked. I commented line-by-line to be sure I understood what was going on. Essentially the program looped through each character entered in the serial and xored with 63, then after that loop, it compared the now-xored characters, 1 at a time in a new loop, with characters at a memory address.
  • My method was to "follow in dump" the referenced memory address, copy the characters down and xor each of them, revealing the correct serial. I didn't have a good utility for simple xor-ing (ahem - we need the ultimate cracker tool!), so I did them one at a time
